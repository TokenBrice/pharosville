# Support

PharosVille support happens through GitHub issues when the report can be public and does not include secrets.

## Use GitHub Issues For

- reproducible app bugs
- visual regressions
- documentation problems
- stablecoin or chain signal questions with source links
- feature requests that explain the user problem

Use the issue forms so reports include viewport, browser, source, and validation context where relevant.

## Use Private Reporting For

- exposed credentials
- auth bypasses
- data-leak risks
- abuse paths
- security-sensitive reproduction details

Follow [SECURITY.md](./SECURITY.md) for vulnerability reporting.

## Scope

PharosVille is an interpretive stablecoin data visualization. It is not financial advice and does not provide wallet, trading, custody, account, or investment support.

The maintained production URL is `https://pharosville.pharos.watch/`.
